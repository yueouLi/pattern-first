# Migration: schema_version 1.3 → 1.4

> **Type**: MINOR **but BREAKING for blind channel integrity**—state fields unchanged, but `rubric_notes.md` must be split. **Before an existing user runs migrate, the blind sub-agent still leaks actuals.**

---

## WHAT changed

### File layer (**core**)

| Before | After |
|---|---|
| `rubric_notes.md` accumulated all bump Memos at the top (with video names + actuals + derived evidence) | `rubric_notes.md` contains only generic language (formula / dimension definitions / bucket boundaries / pointers), **never** contains actuals |
| —— | `rubric-memo.md` (**new**) accumulates the full text of all bump Memos (with video names + actuals + derived evidence) |

### State fields

No additions / removals / renames. Only the `schema_version` bump marks that this "file-layer migration" must run.

### Channel B hard refusal list

`cheat-score-blind/SKILL.md` adds the `rubric-memo.md` path + refusal_code `blocked_rubric_memo`.

---

## WHY

When PR #11 introduced the blind-scoring sub-agent (channel B) it promised it could only read `scripts/<id>.md` + `rubric_notes.md`. But cheat-bump Phase 5, when landing, wrote the upgrade Memo (with the evidence-data table + derived evidence + real video names + actuals) into rubric_notes.md—the sub-agent read the actuals it shouldn't see through the whitelist, and blind scoring became "post-hoc rationalization having seen the actuals".

Reproduced in practice (PR #11 test):

```
contamination_note: rubric_notes.md v0→v1 bump memo contains the same-named entry for this script "the one that got me 380k" actuals
contamination_note: rubric_notes.md derived-evidence section contains this script's name + actual 137k, the CC anchor was pre-calibrated
→ refusal: "non_blind_warning"
→ all dimensions confidence: medium
```

The fix: split files. `rubric_notes.md` strictly holds only generic rules (the blind whitelist), the bump history archive is accumulated into `rubric-memo.md` (blind hard-forbidden to read). See the "Blind channel leak guard" section of [shared-references/observation-lifecycle.md](../shared-references/observation-lifecycle.md).

---

## HOW (Claude steps for /cheat-migrate)

Execute in order. **Non-idempotent steps must first check whether they've already run**—this migration involves content-file rewriting, and re-running may produce duplicate content.

### Step 1: prerequisite checks

1. Read `.cheat-state.json` → verify `schema_version == "1.3"`, otherwise error "the current state isn't 1.3, this migration isn't needed"
2. Check `rubric_notes.md` exists; doesn't exist → error "can't find rubric_notes.md. Has the project not been init'd?"
3. Check **whether `rubric-memo.md` already exists**:
   - already exists → ask the user "rubric-memo.md already has content, skip the split and just bump schema to 1.4? Reply yes to skip, no to abort the migration"
   - doesn't exist → proceed to Step 2

### Step 2: backup

```bash
cp rubric_notes.md rubric_notes.md.backup-$(date +%s)
```

Prompt the user: "backed up `rubric_notes.md`—recoverable if the split errors".

### Step 3: scan + extract the forbidden sections of `rubric_notes.md`

Read the full `rubric_notes.md`. Identify the following patterns:

| Pattern | Handling |
|---|---|
| A whole `## vX → vY upgrade Memo` section with subsections (triggering observation / evidence data / diagnosis / cross-model audit / known limitations) | **extract the whole section** to rubric-memo.md, **delete** in place |
| A derived-evidence section with a video name + numbers (e.g. "stop expecting" actual 137k) | **extract the whole entry** to the "derived evidence" subsection of rubric-memo.md; replace in place with generic language ("high-abstraction-density sample → CC=1 → low reach") |
| Calibration-pool re-score table / ranking-comparison table | **extract the whole table** to the "evidence data" subsection of the corresponding Memo in rubric-memo.md; delete in place (keep the formula quick-reference table—the formula quick-reference contains no video names) |
| Cross-model audit citation with a specific video name + actuals | extract |
| Scattered `\d+w` / `\d+万` numbers (**exception: bucket boundaries** like 50k-300k / 300k-1M etc. are kept) | use Claude's judgment: is it a bucket boundary → keep; is it an actuals citation → extract |

### Step 4: write `rubric-memo.md`

If the file doesn't exist → copy the empty skeleton from `pattern-first/templates/rubric-memo.template.md`.

Then append the Memo sections extracted in Step 3 to the end of the file **in chronological order** (old on top / new on bottom). Wrap each Memo in the 6-component format of [templates/rubric-memo.template.md](../templates/rubric-memo.template.md); mark missing components `(unknown from legacy migration)`.

### Step 5: clean up `rubric_notes.md`

- Add to the top metadata section:
  ```markdown
  **Upgrade memos**: see [rubric-memo.md](rubric-memo.md)
  ```
- Where a Memo section was extracted in Step 3, leave a **short pointer**: "the v0→v1 upgrade Memo has been migrated to rubric-memo.md, run `cat rubric-memo.md` to view"
- Replace the derived-evidence section with generic language

### Step 6: self-check leak guard

Run on the cleaned `rubric_notes.md`:

```bash
grep -E '[0-9]+\s*[wWmMkK万]|plays|actual|实绩' rubric_notes.md \
  | grep -v -E '^\s*\|[^|]*\|[^|]*\|.*[0-9]+\s*[wkM][^a-zA-Z]' \
  | grep -v -E '50k-300k|300k-1M|1M-1.5M|>1.5M'
```

(the grep excludes obvious bucket-boundary numbers)

- No hits → pass
- Hits → report the remaining "actuals citations" + have the user manually review / extract further → continue to Step 7 or abort

### Step 7: bump schema + write back state

1. Set `.cheat-state.json`'s `schema_version = "1.4"`
2. **Atomic write**
3. Report: "✅ migration to 1.4 complete. `rubric_notes.md` now contains only generic language ({N} lines), `rubric-memo.md` contains {M} historical Memo sections. Backup kept: rubric_notes.md.backup-{ts}."

## Manual fallback

Users who don't want to run `/cheat-migrate` can **split manually**:

1. Backup: `cp rubric_notes.md rubric_notes.md.backup`
2. Cut all `## vX → vY upgrade Memo` sections wholesale from `rubric_notes.md` into a **newly-created** `rubric-memo.md` (keep chronological order)
3. Cut the derived-evidence section's video names + actuals into rubric-memo.md, replace in place with generic language
4. Add a pointer `**Upgrade memos**: see [rubric-memo.md](rubric-memo.md)` at the top of `rubric_notes.md`
5. Hand-edit `.cheat-state.json`'s `schema_version` to `"1.4"`

Or the **laziest**: delete `rubric_notes.md` and re-init (loss: all historical observations + bump memos), run `bash install.sh` then say "init"—cheat-init recreates the two empty files. Only suitable for early users whose calibration pool is still small (< 5).
