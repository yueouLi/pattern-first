# Adapter: douyin-session (Douyin scraping)

Auto-called by `/cheat-retro` when `state.data_collection=adapter` + `platform=douyin`.

> **Source**: lifted from the reference creator's project (private) `crawler.py` / `review.py` / `renderer.py`—already validated on a Chinese opinion-video creator's account across 25+ videos.

---

## What this adapter is for

A Douyin video page is **heavily JS-rendered + loads comments via XHR + has severe anti-scraping**—approaches without a browser engine and a login state (e.g. Claude's WebFetch, Python requests) get no data.

douyin-session uses **Playwright + a persistent Chromium context** to simulate a real browser:
- You scan a QR code to log into Douyin Creator Center the first time; the cookie is stored in **your content project's root** `.auth/`
- Each subsequent fetch reuses the cookie directly, no re-login
- Intercepts Douyin's frontend XHR responses to grab the data-endpoint JSON directly (no HTML parsing)
- Fetches 3 kinds of data: video list / detailed data (completion / follow-conversion rate) / comments

Output is written to **your content project's** `videos/<...>/report.md` (`cheat-retro` reads this file → writes a summary into the prediction retro section).
Debug artifacts (URL dumps / screenshots) are written to `.cheat-cache/douyin-session-debug/`, to avoid scattering them in the skill source directory.

## Install (one-time)

```bash
# 1. enter your content project's root
cd ~/my-channel

# 2. create a virtualenv (strongly recommended—Playwright + Chromium are hundreds of MB, don't pollute system Python)
python3 -m venv .venv
source .venv/bin/activate

# 3. install Playwright
pip install playwright>=1.44

# 4. install Chromium (required the first time)
playwright install chromium

# 5. scan-QR login to Douyin Creator Center the first time
ADAPTER=$(find ~/.claude/skills -name "douyin-session" -type d 2>/dev/null | head -1)
# if not found (the adapter may not be global), use the source path:
# ADAPTER="$HOME/Desktop/cheat-test/cheat-on-content/adapters/perf-data/douyin-session"
python "$ADAPTER/crawler.py" login
# → a Chromium window pops up, scan the QR to log into Creator Center
# → after login the window auto-closes, the cookie is stored in current-dir/.auth/
```

## Usage

cheat-retro calls it automatically, you don't need to run it manually. But if you want to test manually:

```bash
cd ~/my-channel
source .venv/bin/activate

# list recent videos (to see whether the login state expired)
python "$ADAPTER/crawler.py" list

# fetch a specific video
python "$ADAPTER/review.py" video <aweme_id> <video_folder>/script.md

# output at current-dir/videos/<date>_<title>/report.md
```

## How to get the aweme_id

Douyin video URL forms:
- `https://www.douyin.com/video/7234567890123456789` → `aweme_id = 7234567890123456789` (directly in the URL path)
- `https://v.douyin.com/abc123` → the short link needs resolving (cheat-publish does it automatically)

cheat-publish stores the aweme_id into the prediction header at publish registration (if it can resolve). cheat-retro reads this field directly on startup.

## report.md output format

Generated by `renderer.py`. Contains:
- Video metadata (title, publish time, duration)
- Data snapshot (plays, likes, comments, shares, saves + derived ratios: like rate / comment rate / share rate)
- Completion rate / 3s retention (if obtainable)
- Top 20 comments (sorted by likes, with comment text + like counts)
- Comment-keyword clustering (renderer does it automatically, optional)

## Failure modes (by probability, high to low)

| Symptom | Cause | Handling |
|---|---|---|
| `ensure_login` timeout | cookie expired or Douyin forced a reauth | re-run `python crawler.py login` |
| `_parse_video_list` returns empty | Douyin changed the endpoint fields—a structural change | look at the URLs captured in `.debug/creator_urls.txt`, update crawler.py's field fallbacks |
| `_parse_video_list` video list incomplete | pagination missed—slow network or anti-scraping triggered | increase the `await asyncio.sleep(...)` duration in `crawler.py` |
| Chromium crash / hang | usually insufficient machine memory | close other Chromium processes; `playwright install chromium --force` to reinstall |
| comment fetch slow (>5min) | the comment page XHR triggers on multiple scrolls—slow network | reduce `fetch_comments_creator`'s `max_pages`, or switch to a more stable network |

**Key reality**: Douyin's endpoint structure changes every few months. This adapter **needs ongoing maintenance**—if one day it suddenly fails, the first step is to check the `video analysis` project's latest crawler.py for a new version.

## Stability level

★★—the Playwright approach withstands far stronger anti-scraping than pure HTTP, but is still affected by Douyin frontend redesigns. Suggest manually running `crawler.py list` once a month to verify health.

## Risk notes

- **Cold-start users beware installing**: Playwright + Chromium are large (~500MB), easily discouraging newcomers
- **TOS risk**: using your own cookie to fetch your own backend data is personal use; don't abuse it
- **Don't commit .auth/ to git**: the cookie contains your session credentials, leaking it = others can log into your Douyin account
- `.cheat-cache/douyin-session-debug/` should also not be committed to git: it may contain debug screenshots and endpoint URL dumps
